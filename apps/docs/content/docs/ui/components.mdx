# Components

## Layout

### `MapShell`
The main layout wrapper. It establishes the "map application" context, positioning the map in the background and managing slots for UI elements.

- **Props**:
  - `sidebar`: Node to render in the collapsible sidebar (usually `LayerPanel`).
  - `toolbar`: Node to render in the toolbar area.
  - `topBar`: Node to render at the top (branding, search).
  - `statusBar`: Node to render at the bottom (coordinates, scale).

### `MapViewport`
A lower-level component that renders the map container div. Use this if you are building a completely custom layout and don't want the shell storage.

## Panels

### `LayerPanel`
A complex, interactive panel for managing layers.
- Drag-and-drop reordering.
- Visibility toggles.
- Opacity sliders.
- Zoom-to-layer actions.

### `LegendPanel`
Displays legends for visible layers. Automatically fetches WMS legends or renders vector styles.

### `FeatureDetailsSheet`
A slide-over sheet that displays attributes of selected features.

## Toolbars

### `MapToolbar`
A strip of tools (`Draw`, `Measure`, `Identify`). It integrates with the `InteractionModeStore` to switch active modes.

### `BasemapSwitcher`
A UI element to toggle between different basemap styles (Satellite, Street, Dark).

## Dialogs

### `AddLayerDialog`
A modal for adding new layers. Supports:
- **WMS/WMTS**: Enter a URL and browse capabilities.
- **GeoJSON**: Upload or paste GeoJSON.
- **PMTiles**: Add vector/raster archives.
